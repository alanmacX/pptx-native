# Bug reports

Self-contained, hand-off-ready bug write-ups (one file per issue). Each should be
readable by a fresh agent with no prior context: symptom, repro, what's been
ruled out, suspected root causes, and concrete next steps. Reference repo-relative
paths only (no machine-specific paths like ~/Desktop).

Naming: `<short-slug>.md` (e.g. `preview-blank.md`).
